# Investigación sobre Conceptos de Sistemas Operativos

## Tipos de Kernel y sus Diferencias

### Monolítico
- **Descripción**: El kernel monolítico es un tipo de kernel en el que todos los servicios del sistema operativo, como la gestión de procesos, memoria y el sistema de archivos, están incluidos en el mismo espacio de usuario.
- **Ejemplos**: Linux, UNIX.
- **Ventajas**: Alta eficiencia debido a la ausencia de la necesidad de contextos de cambio entre el núcleo y el espacio de usuario.
- **Desventajas**: Menos modular y más difícil de mantener. Un fallo en cualquier parte del kernel puede afectar a todo el sistema.
- **Diagrama**:
  ![Monolithic Kernel](https://example.com/monolithic_kernel.png)
  *Fuente: [Example Link](https://example.com)*

### Microkernel
- **Descripción**: En un microkernel, el kernel proporciona solo las funciones básicas y esenciales, como la comunicación entre procesos y la gestión de hardware. Otros servicios del sistema operativo se ejecutan en el espacio de usuario.
- **Ejemplos**: Minix, QNX.
- **Ventajas**: Mejor modularidad y estabilidad, ya que los servicios del sistema son independientes y se pueden reiniciar sin afectar al núcleo.
- **Desventajas**: Puede haber un mayor costo de comunicación entre los componentes del sistema, lo que puede llevar a una menor eficiencia en comparación con los kernels monolíticos.
- **Diagrama**:
  ![Microkernel](https://example.com/microkernel.png)
  *Fuente: [Example Link](https://example.com)*

### Híbrido
- **Descripción**: Los kernels híbridos intentan combinar lo mejor de los kernels monolíticos y los microkernels. Incluyen algunos servicios en el núcleo para mejorar el rendimiento, pero también permiten que otros servicios se ejecuten en el espacio de usuario.
- **Ejemplos**: Windows NT, macOS.
- **Ventajas**: Ofrecen un equilibrio entre eficiencia y modularidad.
- **Desventajas**: La complejidad en la gestión de servicios puede ser mayor.
- **Diagrama**:
  ![Hybrid Kernel](https://example.com/hybrid_kernel.png)
  *Fuente: [Example Link](https://example.com)*

## User Mode vs Kernel Mode

### User Mode
- **Descripción**: Es el modo en el que las aplicaciones y los procesos de usuario se ejecutan. El acceso a los recursos del sistema y a las instrucciones del hardware está restringido para proteger la integridad del sistema operativo y evitar la ejecución de instrucciones que podrían comprometer la estabilidad del sistema.
- **Ventajas**: Mayor seguridad y estabilidad, ya que las aplicaciones no pueden acceder directamente al hardware ni a los recursos críticos del sistema.
- **Diagrama**:
  ![User Mode](https://example.com/user_mode.png)
  *Fuente: [Example Link](https://example.com)*

### Kernel Mode
- **Descripción**: Es el modo en el que el sistema operativo y los controladores de hardware se ejecutan. En este modo, el código tiene acceso completo al hardware y a la memoria del sistema.
- **Ventajas**: Permite al sistema operativo gestionar el hardware directamente y ejecutar instrucciones críticas con alta eficiencia.
- **Diagrama**:
  ![Kernel Mode](https://example.com/kernel_mode.png)
  *Fuente: [Example Link](https://example.com)*

## Interruptions vs Traps

### Interruptions
- **Descripción**: Las interrupciones son señales enviadas por el hardware para indicar que se ha producido un evento que necesita atención inmediata. Pueden ser generadas por dispositivos periféricos, como discos duros o teclados.
- **Ejemplo**: Un teclado enviando una señal cuando se presiona una tecla.
- **Diagrama**:
  ![Interruptions](https://example.com/interruptions.png)
  *Fuente: [Example Link](https://example.com)*

### Traps
- **Descripción**: Los traps (o excepciones) son señales generadas por el software cuando ocurre una condición excepcional durante la ejecución de un programa, como una división por cero o una violación de acceso a memoria.
- **Ejemplo**: Un intento de acceder a una dirección de memoria no válida.
